## I2C
 - ### I2C 是甚麼

      Master-Slave 的架構，Master 可以向 Slave 發出需求要資料或傳送資料，
      在I2C通訊協定中收發資料只靠兩條線SCL(serial clock) 以及 SDA(serial data)，換句話說 **I2C是有線通訊** ，而Master要跟Slave溝通則是透過Address

 - ### I2C 運做原理
    #### I2C 採用主從協定。主機發起通訊，事件的順序如下：
     1. 主機發出起始狀態，這個狀態通知所有裝置，在串行資料線上等待指令
     2. 主機發送目標裝置的位址和讀寫旗標
     3. 具有匹配位址的裝置回應一個確認訊號
     4. 主機和裝置之間的通訊在資料匯流排上進行，無論這個通訊是讀還是寫，主機和裝置都可以接收或傳輸資料，發送器將8位元的資料發送給接收器，接收器則回應1位元的確認訊號
     5. 當通訊完成時，主機發出停止狀態，表示一切都完成了
        > 7Bits Address + 1Bit R/W Bit + 1Bit ACK/NACK Bit

 - ###  I2C缺點
     1. 速度有限
        > 不適合大資料量傳輸
     2. 線路限制
        > SDA / SCL 共用 → 長線路會有訊號衰減

        >上拉電阻 + 線路電容會影響 RC 上升時間 → 限制距離

     3. 複雜度高
        > 需要處理 Address、ACK/NACK、Start/Stop
     4. 硬體限制

## UART
 - ### UART 是甚麼
   UART不同於I2C，UART是靠兩邊 **相同速度** 來傳遞、接收資料，也就是靠 **鮑率(baud rate)**，是一個非同步傳輸

 - ### 如何傳送
   以 9600 baud 的速度來講，理論上每秒可以傳送 9600 個 bit，每個bit的時間就是 1/9600 = 104.16us。

   當UART開始傳送時，會先發送1 bit 的start condition的信號，平時這個UART保持在高電位，當要開始傳送時就讓UART維持低電位，這樣接收端就知道說有資料要傳送過來了，那就以start conditiond往後數104.16us，就是第一個 data bit 的開始

 - ### UART 的通訊規則（協定內容）
   - #### 資料格式
     - Start bit → 1 bit
     - Data bits → 5~8 bit（通常 8 bit）
     - Parity bit → 可選（檢查錯誤）
     - Stop bit → 1~2 bit
   - #### 傳輸速率
     - 雙方必須同意，例如 9600 / 115200 bps
   - #### 雙向通訊
     - TX → 傳送
     - RX → 接收
     - 一般點對點（1對1）
   - #### 非同步特性
     - 收發端不共用時鐘線
     - 每個 byte 由 start bit + stop bit 框起來

 - ###  UART缺點
     1. 只能點對點
        > 1 對 1 傳輸，不能同一條線接多個裝置

        > 設備通訊需要額外硬體或多條 UART
     2. 無位址概念
        > 裝置間無識別碼，容易出現衝突

        >需要軟體協定自行管理訊息

     3. 沒有內建流量控制
        > 大量資料傳輸可能需要額外控制，否則容易丟資料
