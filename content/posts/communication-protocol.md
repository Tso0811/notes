## I2C

### 什麼是 I2C

I2C 採用 **Master-Slave** 架構，只靠兩條線傳輸資料：

| 線路 | 全名 | 用途 |
|------|------|------|
| SCL | Serial Clock | 時鐘訊號（由 Master 控制） |
| SDA | Serial Data | 資料傳輸 |

Master 透過 **Address** 與對應的 Slave 溝通。

### 通訊流程

1. Master 發出**起始訊號**，通知所有裝置等待指令
2. Master 發送目標裝置的 **7-bit 位址** + **1-bit R/W 旗標**
3. 匹配位址的 Slave 回應 **ACK（確認）**
4. 開始傳輸資料：發送器傳 8-bit 資料，接收器回應 1-bit ACK/NACK
5. 通訊完成，Master 發出**停止訊號**

```
封包格式：[ 7-bit Address | R/W bit | ACK/NACK ]
```

### I2C 缺點

| 缺點 | 說明 |
|------|------|
| 速度有限 | 不適合大資料量傳輸 |
| 線路限制 | SDA/SCL 共用，長線路有訊號衰減；上拉電阻 + 線路電容影響 RC 上升時間 |
| 複雜度高 | 需處理 Address、ACK/NACK、Start/Stop 邏輯 |
| 硬體限制 | 裝置數量受位址空間限制 |

## UART

### 什麼是 UART

UART 是**非同步**傳輸協定，不共用時鐘線，而是靠雙方設定**相同的鮑率（Baud Rate）**來同步收發。

> 非同步 = 收發端不共用時鐘線，每個 byte 由 Start bit + Stop bit 框起來。

### 傳輸原理

以 9600 baud 為例：
- 每秒可傳 9600 個 bit
- 每個 bit 的時間 = 1 / 9600 ≈ **104.16 μs**

**傳輸流程：**
1. 平時保持高電位（Idle）
2. 要傳送時拉低電位 → 接收端偵測到 **Start bit**
3. 往後每隔 104.16 μs 取樣一次，依序讀出資料 bit

### 資料格式

```
| Start bit (1) | Data bits (5~8) | Parity bit (可選) | Stop bit (1~2) |
```

| 欄位 | 大小 | 說明 |
|------|------|------|
| Start bit | 1 bit | 固定為低電位，通知接收端開始 |
| Data bits | 5~8 bit | 通常為 8 bit |
| Parity bit | 可選 | 用於錯誤檢查 |
| Stop bit | 1~2 bit | 固定為高電位，表示結束 |

**雙向通訊：**

| 接腳 | 功能 |
|------|------|
| TX | 傳送（Transmit） |
| RX | 接收（Receive） |

### UART 缺點

| 缺點 | 說明 |
|------|------|
| 只能點對點 | 1 對 1，同一條線無法接多個裝置 |
| 無位址概念 | 裝置間無識別碼，需軟體自行管理 |
| 無內建流量控制 | 大量資料可能丟失，需額外機制處理 |

## I2C vs UART 比較

| 特性 | I2C | UART |
|------|-----|------|
| 線數 | 2（SCL + SDA） | 2（TX + RX） |
| 同步方式 | 同步（共用時鐘） | 非同步（鮑率對齊） |
| 多裝置 | ✅ 支援（透過位址） | ❌ 僅點對點 |
| 速度 | 較慢 | 較快 |
| 複雜度 | 較高 | 較低 |
